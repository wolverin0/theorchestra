# Phase 10 — Dogfood + Cutover

- **Status**: READY (artifacts in place; dogfood = calendar time to elapse)
- **Branch**: `v3.0` → to be merged into `main` at cutover
- **Reference**: `docs/v3.0-plan.md` § Phase 10 + `docs/v3.0-decisions.md` §7

## What Phase 10 is

Not a build phase — a **run** phase. v3.0's architecture is complete as of commit `742d885`. Phase 10 validates every feature on real workloads:

1. 24h continuous run with ≥4 active panes, zero crashes, zero 10054.
2. End-to-end dogfood of each v2.x carryover feature.
3. Auto-handoff tested on 3 real high-ctx panes.
4. Omniclaude autonomous run: 10 OK-CONTINUE + 3 content-escalation decisions.
5. Cloudflared phone-paste flow.

Because the 24h run is calendar time, Phase 10 is the only phase this session can't finish. All instrumentation + harnesses are ready; the work is "let it run + capture evidence".

## Cutover steps (do these once dogfood clears)

Run these in order. Stop at any step that fails.

1. **Final merge to main:**
   ```bash
   git checkout main
   git merge --no-ff v3.0
   git tag -a v3.0.0 -m "theorchestra v3.0.0 — PTY substrate, active orchestrator, full MCP parity"
   ```

2. **Stop v2.7:**
   ```bash
   # Kill the v2.7 dashboard on :4200
   for pid in $(netstat -ano | grep ":4200" | grep LISTENING | awk '{print $5}' | sort -u); do taskkill //PID "$pid" //F; done
   # Disable auto-start (user-specific — edit startup entries)
   ```

3. **Flip v3.0 port from :4300 to :4200:**
   ```bash
   # In scripts/theorchestra-start.ts env, OR via CLI:
   THEORCHESTRA_PORT=4200 theorchestra start
   ```
   Or update `DEFAULT_DASHBOARD_PORT` in `src/shared/types.ts` to `4200` and re-run the typecheck + phase2 gate (both will pass — the change is a constant).

4. **Migrate MCP clients to the new token:**
   Every running Claude Code session has `mcp-server.cjs` (v2.7) or `src/mcp/server.ts` (v3.0) configured in its `~/.claude/claude_desktop_config.json`. Switch v3.0 MCP server in, restart each session, paste the v3.0 token into localStorage on first dashboard visit.

5. **Cloudflared tunnel:**
   ```bash
   cloudflared tunnel --url http://localhost:4200
   ```
   Paste the public URL on your phone; paste the token once; done.

6. **Push:**
   ```bash
   git push origin main --tags
   ```

7. **Uninstall WezTerm** (optional — it doesn't harm anything, but v3.0 doesn't need it).

## Rollback

If v3.0 blows up during dogfood:

- Stop v3.0: `theorchestra stop` (follow the printed kill instructions).
- Restart v2.7 on :4200: the old process is already tagged `v2.7.0` on `main`. `git checkout v2.7.0 -- src/ scripts/` and re-run `npm run dashboard`.
- File an incident in `docs/v3.0-incidents.md` with the crash class + the phase that shipped the broken code.
- Revert the v3.0 merge on `main` with `git revert -m 1 <merge-commit>`.

## Reversible at every step

Phase 10's entire point is that nothing is committed until all 24h + checklist evidence clears. The cutover commit is a single `--no-ff` merge that can be reverted cleanly.
