# ADR-002 Addendum — Adopt `@wterm/react` (was briefly `@xterm/xterm`)

- **Date**: 2026-04-20 (REMEDIATION pivot)
- **Status**: Accepted — supersedes ADR-002 §"`@wterm/react` for `<Terminal />`"
- **Parent**: `docs/adrs/v3.0-002-frontend-react-vite.md`
- **Companion**: `docs/REMEDIATION.md`

## What happened

ADR-002 specified `@wterm/react` (vercel-labs/wterm) as the terminal
component. During Phase 1 implementation the frontend landed using
`@xterm/xterm` + `@xterm/addon-fit` instead, with no ADR documenting
the swap. That divergence lived for several weeks and was caught in
2026-04-20 when the user asked "weren't we supposed to use wterm?"

## Why the original pick (`@wterm/react`) is correct

- **DOM-based rendering** (Zig/WASM → DOM, not canvas). You get native
  browser text selection, Ctrl+F find, and a real a11y tree — all
  free. xterm.js renders to canvas and has none of those without
  bolt-ons.
- **A11y tree is the point.** It's the half of the v3.0 spec that
  `agent-browser` needs to observe and act on the dashboard (ADR-003
  + its addendum re-accepted). Canvas xterm.js makes that path
  unavailable.
- **Smaller bundle.** Measured: 484 KB → 234 KB JS (−52%), 133 KB → 76
  KB gzipped.

## What changed in the code

- `npm i @wterm/react @wterm/core`; uninstalled `@xterm/xterm`,
  `@xterm/addon-fit`, `@xterm/addon-serialize`. `@xterm/headless`
  stays — it backs the server-side SSE emitter, different use case.
- `src/frontend/Terminal.tsx` rewritten to use `<Terminal/>` +
  `useTerminal()` from `@wterm/react`. `autoResize` prop replaces
  `FitAddon` + `ResizeObserver`.
- Scrollback replay + exponential-backoff reconnect + WebSocket
  transport are preserved 1:1. Only the rendering surface changed.
- Phase 11 Playwright gate remained 21/21 through the pivot.

## Known issue in `@wterm/react@0.1.9`

The package's own types don't line up: `useTerminal()` returns
`RefObject<TerminalHandle | null>` but `<Terminal>` accepts
`RefObject<TerminalHandle>`. Runtime works; a single cast in
`Terminal.tsx` bridges the gap. Upstream fix pending.
